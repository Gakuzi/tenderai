# 🤖 Умная система управления WhatsApp сессиями

Многопользовательская система для управления множественными WhatsApp сессиями с интеллектуальной обработкой сообщений.

## 🎯 Основные возможности

### ✅ Множественные сессии
- **Поддержка неограниченного количества компаний** - каждая компания клиента имеет свою изолированную сессию
- **Headless режим** - все сессии работают в фоне без открытия окон браузера
- **Долгоживущие сессии** - сессии сохраняются между перезапусками сервера
- **Автоматическое переподключение** - сессии восстанавливаются при сбоях

### 🧠 Умная обработка сообщений
- **Избирательная реакция** - агент реагирует только на инициированные им чаты
- **База поставщиков** - автоматическое принятие чатов от известных поставщиков  
- **Игнорирование спама** - неинициированные чаты игнорируются
- **Webhook уведомления** - гибкая система уведомлений о новых сообщениях

### 🔧 API для интеграции
- **RESTful API** - полный набор эндпоинтов для управления
- **React компонент** - готовый UI компонент для интеграции в интерфейс
- **Real-time мониторинг** - отслеживание статуса всех сессий

## 📡 API Endpoints

### Управление сессиями
- `POST /api/whatsapp/initialize/:companyId` - Инициализация сессии для компании
- `GET /api/whatsapp/status/:companyId` - Проверка статуса сессии
- `GET /api/whatsapp/qr/:companyId` - Получение QR кода для авторизации
- `DELETE /api/whatsapp/disconnect/:companyId` - Отключение сессии

### Работа с сообщениями
- `POST /api/whatsapp/initiate/:companyId` - **Инициирование нового чата** (первое сообщение)
- `POST /api/whatsapp/send/:companyId` - Отправка сообщения в активный чат
- `GET /api/whatsapp/active-chats/:companyId` - Список активных чатов

### Управление поставщиками
- `POST /api/whatsapp/suppliers` - Добавление поставщика
- `POST /api/whatsapp/suppliers/load` - Массовая загрузка поставщиков

### Мониторинг и настройки
- `GET /api/whatsapp/stats` - Статистика всех сессий
- `POST /api/whatsapp/message-handler/:companyId` - Установка webhook обработчика
- `POST /api/whatsapp/cleanup` - Очистка неактивных сессий

## 💡 Примеры использования

### Инициализация сессии
```javascript
const response = await fetch('http://localhost:3002/api/whatsapp/initialize/company-123', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ clientId: 'client-456' })
});
```

### Инициирование чата с поставщиком
```javascript
const response = await fetch('http://localhost:3002/api/whatsapp/initiate/company-123', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    phoneNumber: '+79214962555',
    message: 'Здравствуйте! Нас интересует ваше коммерческое предложение.'
  })
});
```

### Загрузка базы поставщиков
```javascript
const suppliers = [
  {
    id: 'supplier1',
    name: 'ООО "СтройМатериалы"',
    phone: '+79214962555',
    inn: '1234567890',
    contactPerson: 'Иванов И.И.'
  }
];

await fetch('http://localhost:3002/api/whatsapp/suppliers/load', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ suppliers })
});
```

## 🔒 Умная логика обработки

### Когда агент РЕАГИРУЕТ на сообщение:
1. **Инициированные чаты** - агент сам начал переписку
2. **Известные поставщики** - номер есть в базе поставщиков  
3. **Автоматически принятые** - чаты, принятые по дополнительным правилам

### Когда агент ИГНОРИРУЕТ сообщение:
1. **Неинициированные чаты** - агент не начинал переписку
2. **Неизвестные номера** - нет в базе поставщиков
3. **Спам и реклама** - сообщения от незнакомых контактов

## 🔧 Конфигурация

### Переменные окружения
```bash
# В .env файле
WHATSAPP_SERVER_PORT=3002
WHATSAPP_HEADLESS=true
WHATSAPP_SESSION_PATH=.wwebjs_auth
```

### Настройки сессий
- **Headless режим** - включен по умолчанию
- **Timeout подключения** - 60 секунд
- **QR код действует** - 5 минут
- **Очистка неактивных сессий** - 24 часа

## 🚨 Важные особенности

### ⚠️ Ограничения WhatsApp
- Максимум 4 активных устройства на аккаунт
- Сессии могут потребовать периодической реавторизации
- WhatsApp может заблокировать аккаунт при подозрительной активности

### 💡 Рекомендации
- Используйте отдельные номера для каждой компании
- Не отправляйте больше 100 сообщений в час
- Регулярно проверяйте статус сессий
- Настройте мониторинг и уведомления

## 📈 Масштабирование

### Для больших нагрузок:
1. **Кластеризация** - запуск нескольких инстансов сервера
2. **Балансировка** - распределение компаний по серверам
3. **Очередь сообщений** - Redis/RabbitMQ для обработки
4. **База данных** - PostgreSQL для хранения сессий

### Мониторинг производительности:
- CPU и память для каждой сессии
- Количество обрабатываемых сообщений
- Время отклика API
- Статус подключений к WhatsApp
