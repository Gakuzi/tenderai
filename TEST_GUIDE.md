## 🚀 Как протестировать

### Шаг 1: Подготовка компании в базе данных

Сначала убедитесь, что в базе есть компания с ID `comp1`:

```bash
npm run db:seed  # Создаст тестовые данные включая компанию
```

### Шаг 2: Открыть веб-интерфейс для тестирования

Откройте файл в браузере:
```
public/integration-test.html
```

Или запустите простой сервер:
```bash
python3 -m http.server 3000
# Затем откройте http://localhost:3000/public/integration-test.html
```

### Шаг 3: Тестирование WhatsApp Web

1. На странице тестирования нажмите "Подключить WhatsApp"
2. Откроется браузер с WhatsApp Web и сгенерируется QR-код
3. Откройте WhatsApp на телефоне → Меню (⋮) → "Связанные устройства" → "Привязать устройство"
4. Отсканируйте QR-код с экрана
5. Если авторизация прошла успешно, появится форма для отправки сообщений
6. Введите номер телефона (формат: +79991234567) и текст сообщения
7. Нажмите "Отправить" - сообщение должно появиться в WhatsApp

### Шаг 4: Тестирование Telegram Web

1. Нажмите "Подключить Telegram"
2. Откроется браузер с Telegram Web и сгенерируется QR-код
3. Откройте Telegram на телефоне → Настройки → Устройства → "Привязать устройство"
4. Отсканируйте QR-код с экрана
5. Появится форма для отправки сообщений
6. Введите username или имя контакта и текст сообщения
7. Нажмите "Отправить" - сообщение должно отправиться через Telegram

### Шаг 5: Программное тестирование

Для тестирования через код запустите:

```bash
npx tsx test-integrations.ts
```

Этот скрипт:
- Создаст интеграции для WhatsApp и Telegram
- Сгенерирует QR-коды (нужно сканировать вручную)
- Покажет статус подключений

## 🐛 Возможные проблемы

**QR-код не генерируется:**
- Проверьте, что Puppeteer установлен: `npm install puppeteer`
- Убедитесь, что нет запущенных браузеров Chrome

**Не получается отсканировать QR:**
- QR-коды имеют срок жизни 5 минут
- Попробуйте перегенерировать QR-код

**Сообщения не отправляются:**
- Проверьте, что сессия активна (статус CONNECTED)
- Убедитесь, что контакт/номер существует в списке контактов

**Ошибки браузера:**
- На macOS может потребоваться разрешение на запуск браузера
- Попробуйте запустить с `headless: false` для отладки

## 📊 Проверка логов

Все действия логируются в базу данных:

```sql
SELECT * FROM integration_logs ORDER BY createdAt DESC LIMIT 10;
```

Статус интеграций:

```sql
SELECT id, status, displayName, lastCheckedAt, messagesSent 
FROM whatsapp_integrations 
UNION ALL
SELECT id, status, displayName, lastCheckedAt, messagesSent 
FROM telegram_integrations;
```
